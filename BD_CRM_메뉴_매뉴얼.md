# BD CRM v2.0 메뉴 매뉴얼 (팀장 검토용)

> 이 문서는 BD CRM 웹앱의 메뉴별 목적과 실제 동작 방식을 정리한 것입니다.
> 각 항목별로 "목적이 메뉴명과 맞는지 / 필요한 메뉴인지 / 추가로 필요한 기능은 없는지"를 팀장님과 함께 검토하는 용도로 작성했습니다.
> 맨 아래 **[검토가 필요한 항목]** 섹션에 코드 확인 중 발견한 이슈를 별도로 정리해두었습니다.

---

## 0. 화면 상단 요약 카드

페이지를 열면 항상 보이는 5개 지표. "전체 CRM 분석 실행"을 눌러야 최신값으로 갱신됩니다.

| 지표 | 목적 |
|---|---|
| 총 회원수 | 원본_DB에 있는 전체 주문자(고유 ID) 수 |
| 총 주문건수 | 원본_DB의 전체 주문 행 수 |
| 관리 상품수 | 재구매/추가구매 분석 대상으로 등록된 상품 개수 (현재 10개) |
| 재구매 회원수 | **동일 상품을 2회 이상 구매한** 회원 수 |
| 추가구매 회원수 | 서로 다른 상품을 2개 이상 구매한 회원 수 |

## 1. 운영 데이터

| 메뉴 | 목적 | 사용법 / 주의사항 |
|---|---|---|
| 원본 데이터 반영 | `원본_업로드` 시트에 붙여넣은 신규 주문 데이터를 `원본_DB`로 이관(누적) | 반영 후 `원본_업로드` 시트의 데이터 행은 비워짐. 분석 실행 전 단계 |
| 전체 CRM 분석 실행 | `원본_DB` 전체를 기준으로 재구매/추가구매/일자별 흐름 등 **모든 통계를 재계산**해서 각 결과 시트에 반영 | 데이터가 많을수록(현재 약 2만 건) 시간이 걸림(현재 기준 약 20~30초). 새 데이터 반영 후 반드시 눌러야 통계가 최신화됨 |
| 새로고침 | 이미 계산되어 있는 상단 요약 지표만 다시 불러옴 (재계산 아님) | 계산 자체를 다시 하지 않으므로 빠름 |

## 2. 통계 리포트

| 메뉴 | 목적 | 판단 기준 |
|---|---|---|
| 상품별 통합 통계 | 상품별 구매고객수·재구매·추가구매·이탈을 한 표에서 비교 (기존 "상품별 재구매 통계" + "상품별 추가구매 통계" 2개 메뉴를 통합) | 재구매: **동일 상품 2회+ 구매**(시간 제한 없음). 추가구매: 기준상품 외 다른 상품도 보유. 이탈: 1회만 구매하고 재구매도 추가구매도 없음. 재구매와 추가구매는 동시에 해당 가능(중복 O), 이탈은 재구매·추가구매 어느 쪽에도 안 겹침 |
| 상품 이동 전환 Matrix | "A 상품 구매 → B 상품으로 전환"된 고객수를 상품 간 이동 경로로 확인 | 기준상품 최초구매 이후 다른 상품을 구매한 이력 기준 |
| 일자별 전환 흐름 | 날짜별로 고객 구매 흐름을 집계. **2가지 집계 기준**을 선택 가능 | 아래 참고 |
| ㄴ 구매일 기준 | 그날 실제 구매한 고객을 첫구매 / 재구매만 / 추가구매만 / 재구매+추가구매로 분류 | 구매 발생일 기준 |
| ㄴ 재구매 도래일 기준 | 과거 구매일 + 상품별 재구매 예상 주기(옵션의 "OO포" 수량 또는 기본값, **실제 구매수량만큼 배수 적용**)로 계산된 "판정일"에, 실제로 재구매/추가구매/미구매했는지 판정 | 첫구매 개념 없음 (판정대상/재구매완료/추가구매만/미구매/판정대기로 분류) |
| 전체 요약 리포트 | 대시보드 핵심 지표(상단 요약 카드와 동일 데이터)를 표 형태로 조회 | - |

## 3. CRM 대상자 조회

마케팅/CS 액션이 필요한 고객 리스트를 뽑는 용도.

| 메뉴 | 목적 | 입력값 |
|---|---|---|
| 재구매 대상자 조회 | 특정(또는 전체) 상품에서, 마지막 구매 후 일정 기간이 지난 "재구매 유도가 필요한 고객" 리스트 | 상품 선택(미선택 시 전체), 최종 구매 후 경과일(예: 30 → 30일 **이상** 지난 고객만 표시) |
| 추가구매 대상자 조회 | 기준 상품 구매 고객 중, 다른 상품 추가구매 여부와 **추천상품** 확인 | 기준 상품 선택(미선택 시 전체) |

> 참고: "추천상품"은 별도의 추천 로직이 아니라, **관리 상품 10개 중 기준상품을 제외하고 아직 안 가진 상품 전체**를 나열하는 단순 방식입니다.

## 4. 고객 데이터 조회

| 메뉴 | 목적 | 입력값 |

## 검토가 필요한 항목 (팀장 논의용)

코드를 확인하는 과정에서 발견한, 메뉴명과 실제 동작이 다르거나 미완성인 부분입니다.

1. **캠페인 성과 분석 / 성과 조회는 아직 미구현 상태입니다.**
   실제 코드에 `// 성과 분석 로직은 캠페인 기준 정의 후 확장 필요`라는 주석과 함께, 캠페인 목록을 그대로 복사해 보여주기만 합니다. 재구매율·추가구매율 같은 실제 성과 수치는 계산되지 않습니다.
   → **논의 필요:** "성과"를 무엇으로 정의할지(캠페인 기간 중 재구매 전환율? 캠페인 대상상품의 매출 증가율?) 팀장님과 기준을 정해야 실제 기능 개발이 가능합니다.

2. **"일자별 전환 흐름 - 재구매 도래일 기준"에는 "첫구매" 분류가 없습니다.**
   "구매일 기준" 모드는 첫구매/재구매만/추가구매만/재구매+추가구매로 나뉘지만, "재구매 도래일 기준"은 판정대상/재구매완료/추가구매만/미구매/판정대기로 구조가 다릅니다(도래일 판정은 "이미 산 적 있는 사람"만 대상이라 첫구매 개념이 원래 없음).
   → **논의 필요:** 이대로 둘지, 아니면 이 모드에도 추가 분류가 필요한지.

3. **"상품 이동 전환 Matrix 조회" 등 다른 리포트를 볼 때, 화면 상단에 "일자별 전환 흐름" 전용 컨트롤(집계 기준/조회 버튼)이 같이 보이던 문제는 오늘 수정했습니다.** (이제 다른 리포트를 볼 때는 자동으로 숨겨짐)

4. **"재구매 대상자 조회"의 "최종 구매 후 경과일"은 "이상(≥)" 조건입니다.**
   30을 입력하면 "30일 이상 지난" 고객만 나옵니다. 팀장님이 이 숫자를 다르게 해석하실 수 있어(예: "정확히 30일"), 화면에 이 조건을 명시하는 게 좋을지 검토가 필요합니다.

## 메모: 오늘(2026-07-07) 함께 반영된 수정 사항
- 대시보드 "재구매 회원수" 집계 기준을 "생애 주문 2건 이상"에서 "**동일 상품 2건 이상**"으로 수정
- "재구매 도래일" 계산 시 실제 구매수량을 반영하도록 수정 (예: 100포 2개 구매 시 200일 기준으로 판정)
- "일자별 전환 흐름" 관련 화면 제목 문구 통일
- 전반적인 속도 개선 (일자별 전환 흐름 27초→2초, 페이지 로딩 7.7초→0.9초, 전체 분석 실행 55초→24초, 고객 검색 즉시 조회)
- 다른 리포트 조회 시 "일자별 전환 흐름" 전용 컨트롤이 잘못 보이던 화면 버그 수정
- "상품별 재구매 통계"에 재구매횟수 컬럼 추가, "상품별 추가구매 통계"에 추가구매 상품수 컬럼 추가
- "상품별 재구매 통계" + "상품별 추가구매 통계"를 "상품별 통합 통계" 1개 메뉴로 병합, 이탈고객수/이탈율 추가
